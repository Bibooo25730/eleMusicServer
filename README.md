### 请求正文未定义

> 正是每当你需要从客户端接受一些数据到我们的服务器时，我们需要使用正文解析器，以便可以解析请求体的流。
> 我们需要中间件并表达为我们提供 Json 对象的中间件。我们可以从客户端获取它。


### 接口规范

#### 请求方式
- POST 新增
- delete 删除
- put 修改
- get 查询

#### 请求体

-数据格式：除 get 都以 json ，除非批量操作，都限制在单层结构
-字段名：字段名与数据库字段名一致，非数据库字段以 .temp_作为前缀
-字段值：字段值一采用字符串

#### url

/子系统/模块名/资源名
